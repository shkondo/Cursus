# 思考時間 (think time) の役割

## 疑問

visualizer で確認すると、think の時間がほぼゼロに見える。
eat と sleep だけで十分に回るのでは？

## 結論

偶数人数では think は実質不要。奇数人数では飢餓防止のために必要。

## 偶数の場合: think 不要

4人の例 (eat=200, sleep=200):

```text
時間(ms)  0     200    400    600    800
哲学者1:  [食事]  [睡眠]  [食事]  [睡眠]  [食事]
哲学者2:  [睡眠]  [食事]  [睡眠]  [食事]  [睡眠]
哲学者3:  [食事]  [睡眠]  [食事]  [睡眠]  [食事]
哲学者4:  [睡眠]  [食事]  [睡眠]  [食事]  [睡眠]
```

偶数/奇数の2グループに分かれ、一方が食事中にもう一方が睡眠。
フォーク競合が起きないので、sleep から起きたら即座に食事できる。

コードでも偶数なら即 return している:

```c
static void  philo_think(t_philo *philo)
{
    print_status(philo, MSG_THINK);
    if (philo->table->nb_philo % 2 == 0)
        return ;             // ← 偶数: think 時間なし
    ...
}
```

## 奇数の場合: think が必要

### なぜ問題が起きるか

5人の場合、同時に食べられるのは最大 2人 (= (5-1)/2)。
5人を2枠で回す必要があるので、eat + sleep だけでは足りない。

think なしだと (eat=200, sleep=200):

```
時間(ms)  0     200    400    600
哲学者1:  [食事]  [睡眠]  [食事]  ← sleep 直後にすぐ食べられる
哲学者3:  [食事]  [睡眠]  フォーク待ち... ← 1 に先を越された
哲学者5:  待ち    待ち    待ち    ← いつまでも食べられない (飢餓)
哲学者2:  [睡眠]  [食事]  [睡眠]
哲学者4:  [睡眠]  [食事]  [睡眠]
```

sleep から起きた哲学者が即座にフォークを取ると、特定の哲学者が
繰り返し後回しにされ、`time_to_die` 以内に食べられず死亡する。

### think で解決する仕組み

```c
think_time = philo->table->time_to_eat * 2 - philo->table->time_to_sleep;
```

この式の考え方:

- 5人で2枠 → 全員に回るには約2ラウンド (= `time_to_eat * 2`) 必要
- すでに `time_to_sleep` 分は待っている
- 残りの待ち時間 = `time_to_eat * 2 - time_to_sleep`

例: eat=200, sleep=200 → think = 200*2 - 200 = **200ms**

```
時間(ms)  0     200    400    600    800    1000   1200
哲学者1:  [食事]  [睡眠]  [思考]  [食事]  [睡眠]  [思考]  [食事]
哲学者2:  [睡眠]  [食事]  [睡眠]  [思考]  [食事]  [睡眠]  [思考]
哲学者3:  [思考]  [睡眠]  [食事]  [睡眠]  [思考]  [食事]  [睡眠]
哲学者4:  [食事]  [思考]  [睡眠]  [食事]  [思考]  [睡眠]  [食事]
哲学者5:  [睡眠]  [思考]  [睡眠]  [食事]  [睡眠]  [思考]  [食事]
```

全員が均等な間隔で食事でき、飢餓が発生しない。

### 式が 0 以下になるケース

`time_to_sleep >= time_to_eat * 2` の場合、think_time <= 0 になる。
sleep 時間が十分に長いので、待たなくても全員に順番が回る。

```c
if (think_time > 0)
    precise_sleep(think_time, philo->table);
```

## まとめ

| 人数 | think の役割 | 理由 |
| --- | --- | --- |
| 偶数 | 不要 (0ms) | 2グループで完全に交互に食事できる |
| 奇数 | 順番待ち | 同時に食べられる人数 < 全員なので、待ちが必要 |

think は「次の食事まで譲る」時間。偶数なら譲る必要がないからゼロ、
奇数では飢餓を防ぐためにスケジュールの隙間を埋める役割を持つ。
